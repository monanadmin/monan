Advection for ice thickness and tracers.

Author
William Lipscomb
Date
November 2015

This routine (1) computes new values of ice thickness and tracers under horizontal advection, (2) applies surface and basal mass balance terms, and (3) remaps tracers back onto the standard vertical grid. Based on subroutine li_tendency_thickness by Matthew Hoffman Note: The thickness and tracer fields must have had halo updates before calling this subroutine.

Parameters
[in]dtInput: time step (s)
[in]meshpoolInput: mesh information
[in]advecttracersinInput: if true, then advect tracers as well as thickness
[in,out]velocitypoolInput/output: velocity information
[in,out]geometrypoolInput/output: geometry information to be updated
[in,out]thermalpoolInput/output: temperature/enthalpy information to be updated
[in,out]scratchpoolInput/output: work arrays for tracer advection
[out]errOutput: error flag

Compute layer normal velocity and diagnose the advective CFL limit.

Author
William Lipscomb
Date
January 2016

This routine computed layer normal velocities on each edge in preparation for advection. Given these velocities, it diagnoses the advective CFL limit for the edges on this block. The logic here was originally part of subroutine advect_thickness_tracers_upwind, but was moved to a separate subroutine so that an adaptive timestep could be diagnosed at the start of the time step.

Parameters
[in]meshpoolInput: mesh information
[in]normalvelocityInput: normal velocity on cell edges
[out]layernormalvelocityOutput: normal velocity on cell edges, averaged to layer midpoints
[out]minofmaxallowabledtOutput: maximum allowable dt based on CFL condition
[out]errOutput: error flag
